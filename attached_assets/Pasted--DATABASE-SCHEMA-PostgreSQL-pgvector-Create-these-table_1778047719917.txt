## DATABASE SCHEMA (PostgreSQL + pgvector)

Create these tables:

**resumes** — id, user_id, raw_text, file_url, created_at, iteration (int)
**resume_versions** — id, resume_id, version_number, content, score, created_at
**job_descriptions** — id, title, company, raw_text, required_skills (jsonb), created_at
**extracted_skills** — id, resume_id, skill_name, category, confidence_score, embedding (vector 1536)
**skill_gaps** — id, resume_id, job_id, missing_skill, importance_level, suggestion
**validations** — id, resume_id, ats_score, checklist_results (jsonb), feedback

## AI AGENT PIPELINE (LangChain)

Build 5 independent LangChain agents chained together. Each should be a class in `/backend/agents/`:

### 1. ExtractionAgent
- Input: raw resume text
- Extract: skills, experience (years), job titles, education, tools/technologies
- Output: structured JSON with categories (technical, soft skills, domain expertise)
- Use GPT-4o with a detailed system prompt for structured output

### 2. GapAnalysisAgent
- Input: extracted skills JSON + job description text
- Use pgvector cosine similarity to find semantically missing skills
- Output: list of gaps with importance_level (critical/moderate/nice-to-have)

### 3. SuggestionAgent
- Input: skill gaps + resume context
- Output: 3–5 specific, actionable improvement suggestions per gap
- Include: what to add, how to phrase it, where in resume to place it

### 4. RewriteAgent
- Input: original resume + suggestions
- Output: fully rewritten resume text applying all suggestions
- Preserve original tone, just enhance with better keywords and structure

### 5. ValidationAgent
- Input: rewritten resume + job description
- ATS scoring algorithm: keyword density (30%), formatting score (20%), skill match (50%)
- Output: numeric ATS score (0–100) + checklist (jsonb) with pass/fail items

## BACKEND API ROUTES (FastAPI)

POST   /api/resume/upload         — accepts PDF/DOCX, stores, triggers Celery pipeline
GET    /api/resume/{id}/status    — returns pipeline stage progress (SSE stream)
GET    /api/resume/{id}/extraction
GET    /api/resume/{id}/gaps
GET    /api/resume/{id}/suggestions
GET    /api/resume/{id}/rewrite
GET    /api/resume/{id}/diff      — returns before/after diff using diff-match-patch
GET    /api/resume/{id}/validation
POST   /api/resume/{id}/iterate   — trigger a new pipeline run (for iterative improvement)

Use Server-Sent Events (SSE) for real-time pipeline progress updates to the frontend.

## ASYNC PIPELINE (Celery)

When a resume is uploaded:
1. Store resume in DB, return resume_id immediately
2. Kick off Celery chain: extract → gap → suggest → rewrite → validate
3. Each step updates `pipeline_stage` in Redis
4. Frontend polls SSE endpoint to show live progress
5. Each agent result stored to DB as it completes (not waiting for full chain)
